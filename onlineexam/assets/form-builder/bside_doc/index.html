What is this?

BSIDE is an opensource responsive Form builder and an easy to use tool to create all kind of forms and export it as HTML, JSON or zip package.

You can design and create custom forms and layouts just using mouse clicks and no need to write code in HTML or Javascript.

It has a very neat and structured source code so you can easily add it as part of a bigger software.

you can create and publish forms without writing a single line of code.

 

 

 

Features

  • Custom Colors
  • Custom CSS
  • zip File Creation
  • JSON/HTML Export
  • Fully responsive forms
  • custom layouts
  • Nested form/panels
  • Several components such as textbox, textarea, slider, button, paragraph, link and etc

Form action

It specifies Where to send the form-data when the form is submitted.

Possible values are :

  • An absolute URL - points to another web site (like "http://www.example.com/example.htm")
  • A relative URL - points to a file within a web site (like "example.htm")

Or if you don't know how to use it just leave it blank.

Form method

The form method attribute specifies how to send form-data (the form-data is sent to the page specified in the action attribute).

The form-data can be sent as URL variables (with method="get") or as HTTP post transaction (with method="post").

Possible values are 'POST' and 'GET'

If you don't know which one to use just leave it blank. (default value is get).

Form encoding type

The enctype attribute specifies how the form-data should be encoded when submitting it to the server.

Note: The enctype attribute can be used only if form method is "post".

Possible values are:

  • application/x-www-form-urlencoded (default value)
  • multipart/form-data
  • text/plain

application/x-www-form-urlencoded (default value):

Default. All characters are encoded before sent (spaces are converted to "+" symbols, and special characters are converted to ASCII HEX values)

multipart/form-data:

No characters are encoded. This value is required when you are using forms that have a file upload control

text/plain:

Spaces are converted to "+" symbols, but no special characters are encoded.

NOTE: If you don't know which value to set just remember two things:

1- if you have fileupload component in your form then use 'multipart/form-data'.

2 - If you don't have fileupload control and you don't know which value to use then use the default value.

Textbox

Textbox is one of the most important component of every form builder software.

It can be used to insert name, lastname, address or any information from user.

Textboxes in BSIDE consists of two parts: a label and an input.

Label is the caption of the textbox and the input is where user enters the information.

here are the attributes of a textbox in bside:

Value

This is the default value of the textbox. When you publish a form, this value will appear in the textbox as default.

Place holder

The placeholder attribute specifies a short hint that describes the expected value of an input field (e.g. a sample value or a short description of the expected format).

The short hint is displayed in the input field before the user enters a value.

Name

he name attribute specifies the name of an <input> element.

The name attribute is used to reference elements in a JavaScript, or to reference form data after a form is submitted.

Note: Only form elements with a name attribute will have their values passed when submitting a form.

You must fill this attribute in order to get the data when user submit the form.

Has icon?

specify whether the input has icon or not.

Input icon

This is the icon you want to show in the input box. Type the name of the icon (font-awsome icon name) or click the gear icon in this field and choose an icon from icon picker.

Note: The icon will appear in the inputbox if you check the 'Has icon?' property.

  • large

  • xxxlarge

Specifies the background color of the input box. Enter a color name like 'red' , 'blue' or a color code like '#23AE45' or just use the colorpicker by clicking on the gear icon.

Seperator of the caption of the textbox. Default value is ':'

Export Output

In BSIDE, we have four kinds of outputs. like this:


The Export button will create HTML output of the form but it is useless without css files, so you must copy the css folder wherever you want to use the html content.

Another type of output is 'Export Component HTML'. You can export the HTML code of each element by selecting it (just click on the element) and then pressing this button.

If you need a whole package of the form including font-awsome and css files to upload it directly on the server, then 'download zip package' is good enough for you. To use this button you have to setup BSIDE on a php server (Host or something ) with a zlib enabled library on it.

The last type of output is 'Export JSON' output. This output is mostly for developers who want to store the form structure in the database and restore it later. You can restore a form using it's json by calling load_from_json function and pass the JSON as parameter and get the HTML code of the form. Read more about this in developers section.

For Developers

In BSIDE, we have three function you can call and get the appropriate output:

save_to_html()

Call this function to get the HTML output of the whole form you have created.

html_from_json(json_code)

If you have a json output of a form and you want to convert it to html, then this is the function you need to call. json_code is the json string you pass to the function and get the html equivalent of that json string.

save_to_json()

Call this function to get the json output of the form you 've created.

Custom button
